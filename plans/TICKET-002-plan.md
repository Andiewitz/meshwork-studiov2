# Implementation Plan: TICKET-002 (Passport User Type Extension)

## Decision & Strategy
**Refactor the Type Safety**. Instead of relying on `as any` casts to bypass TypeScript checks, we will create a global module augmentation file (`server/types/express.d.ts`). This file will extend the generic `Express.User` interface by inheriting all properties from the `DbUser` shape generated by Drizzle ORM in `shared/schema.ts`.

## Exact Changes Required

### 1. Create Global Declaration File
Create a new file `server/types/express.d.ts`:

```typescript
import { User as DbUser } from "@shared/schema";

// Expand the Express namespace to merge our database User type into the Passport User.
declare global {
  namespace Express {
    // Extending the empty User interface exported by @types/passport
    interface User extends DbUser {}
  }
}
```

### 2. Update tsconfig.json (Verify)
Since `tsconfig.json` includes `server/**/*`, this `.d.ts` file will automatically be picked up by the compiler. No direct changes to `tsconfig.json` are necessary.

### 3. Remove all `(req.user as any)` Explicit Casts
A comprehensive search and replace across the backend routes to utilize proper, safe types:

**In `server/modules/workspace/routes.ts` (11 occurrences):**
*   *Before*: `const userId = (req.user as any)?.id;`
*   *After*: `const userId = req.user?.id;` (Since `isAuthenticated` middleware ensures `req.user` exists, we can safely access `id`).

**In `server/modules/canvas/routes.ts` (3 occurrences):**
*   *Before*: `const userId = (req.user as any)?.id;`
*   *After*: `const userId = req.user?.id;`

**In `server/modules/ai/routes.ts` (5 occurrences):**
*   *Before*: `const userId = (req.user as any).id;`
*   *After*: `const userId = req.user!.id;` (or `req.user?.id`).

## Testing Strategy
1.  Run the TypeScript compiler via `npm run check`. If the solution is successful, there will be absolutely zero squiggly lines or `TS` build failures complaining about `property 'id' does not exist on type 'User'`.
2.  Do a regression test of the API logic using `npm run test:auth`.

## Risk / Tradeoffs
*   **Trade-off**: Requires that developers remember to update `@shared/schema` whenever a new field is attached to users, but since Drizzle ORM requires this anyway, it's essentially zero overhead. 
*   This approach completely eliminates manual casting bugs down the road.
